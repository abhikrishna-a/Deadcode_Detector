# Complete AWS + Vercel Deployment Guide

## Overview

The repo already contains:
- `docker-compose.prod.yml` — production compose file (Django, Celery, RAG, Nginx)
- `nginx/nginx.prod.conf` — API-only nginx config (no frontend upstream)
- `scripts/deploy.sh` — deployment script (pull → copy nginx → build → up → migrate → verify)
- `.github/workflows/cd.yml` — CD workflow (triggers after CI passes, SSHs into EC2)
- `.github/workflows/ci.yml` — CI workflow (lint, test, build check)

You only need **AWS infrastructure**. The code is ready.

### 11a. Lambda + API Gateway (GitHub Webhook → Auto-Analyze)

**Purpose:** When you push to `main`, Lambda receives the GitHub webhook and triggers analysis automatically.

**Step 1: Create Lambda Function**

1. Search → **Lambda** → **Create function**
2. "Author from scratch"
3. Function name: `deadcode-webhook`
4. Runtime: **Python 3.13**
5. Architecture: **x86_64**
6. Permissions: **Create a new role with basic Lambda permissions**
7. Click **Create function**
8. In the code editor, delete all existing code → paste:

```python
import json, hmac, hashlib, urllib.request

GITHUB_SECRET = "changeme-webhook-secret"
EC2_API = "http://<EC2_IP>/api/analyze/"

def lambda_handler(event, context):
    sig = event["headers"].get("x-hub-signature-256", "")
    expected = "sha256=" + hmac.new(
        GITHUB_SECRET.encode(), event["body"].encode(), hashlib.sha256
    ).hexdigest()
    if not hmac.compare_digest(sig, expected):
        return {"statusCode": 401, "body": "Invalid signature"}

    payload = json.loads(event["body"])
    if payload.get("ref") != "refs/heads/main":
        return {"statusCode": 200, "body": "Not main branch"}

    data = json.dumps({"repo_url": payload["repository"]["clone_url"]}).encode()
    req = urllib.request.Request(EC2_API, data=data, headers={"Content-Type": "application/json"})
    urllib.request.urlopen(req)
    return {"statusCode": 200, "body": "Analysis queued"}
```

Replace `<EC2_IP>` with your instance IP.

9. Click **Deploy**

**Step 2: Add API Gateway Trigger**

1. Click **"Add trigger"** (above the code editor)
2. Select: **API Gateway**
3. Create a new API:
   - API type: **REST API**
